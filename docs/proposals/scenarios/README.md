# Ianvs Scenario Proposal Phase Overview

## Purpose

This document organizes the scenario proposals under `docs/proposals/scenarios` into three development phases so contributors can understand the recommended delivery order.

## Phase Summary

| Phase | Name | Main Focus |
| --- | --- | --- |
| [Phase 1](phase-1-foundation/README.md) | Foundation and Example Maintainability | Example restoration, validation readiness, and maintainability. |
| [Phase 2](phase-2-core-benchmark/README.md) | Core Benchmark Capability | Reusable LLM and edge benchmark infrastructure. |
| [Phase 3](phase-3-advanced-scenarios/README.md) | Advanced Scenarios and Vertical Applications | Robotics, industrial intelligence, privacy, VLA, physical scene generation, and domain applications. |

## Recommended Reading Order

1. [Phase 1: Foundation and Example Maintainability](phase-1-foundation/README.md)
2. [Phase 2: Core Benchmark Capability](phase-2-core-benchmark/README.md)
3. [Phase 3: Advanced Scenarios and Vertical Applications](phase-3-advanced-scenarios/README.md)

## Dependency Direction

Phase 1 establishes a more reliable example base and maintainability direction for Ianvs proposals.

Phase 2 builds reusable benchmark interfaces, execution patterns, and reporting capabilities on top of that foundation.

Phase 3 extends those capabilities into more application-oriented and research-oriented scenarios that depend on clearer validation workflows and stronger benchmark infrastructure.
